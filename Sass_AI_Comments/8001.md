### Configure your setup for port 8001 (fix CORS and base URL)

You confirmed your API runs on 8001. Do these two things so booking from the Professionals page works:

#### 1) Point the Admin UI to the correct API base
- Create or update `admin_ui/.env` with:
  ```bash
  VITE_API_BASE=http://127.0.0.1:8001/v1
  ```
- Restart Vite (env changes require restart):
  ```bash
  cd /Users/rahmath.shihabudeen/PycharmProjects/SaasProject/admin_ui
  npm run dev
  ```

#### 2) Allow your Vite origin(s) via CORS on the backend
- FastAPI reads `CORS_ORIGINS` from env (see `app/main.py`). Start your API with both localhost and 127.0.0.1 Vite origins allowed:

macOS/Linux (zsh/bash):
```bash
export CORS_ORIGINS="http://localhost:5173,http://127.0.0.1:5173,http://localhost:5174,http://127.0.0.1:5174"
uvicorn app.main:create_app --factory --host 127.0.0.1 --port 8001 --reload
```

Windows (PowerShell):
```powershell
$env:CORS_ORIGINS = "http://localhost:5173,http://127.0.0.1:5173,http://localhost:5174,http://127.0.0.1:5174"
uvicorn app.main:create_app --factory --host 127.0.0.1 --port 8001 --reload
```

Notes:
- Keep the host as `127.0.0.1` to match the base URL you set for the UI.
- If you browse Vite using `http://localhost:5173`, keeping both `localhost` and `127.0.0.1` in `CORS_ORIGINS` avoids surprises.

#### 3) Quick CORS preflight test (optional but useful)
Replace port if needed:
```bash
curl -i -X OPTIONS "http://127.0.0.1:8001/v1/tenants/ss-salon/appointments" \
  -H "Origin: http://localhost:5173" \
  -H "Access-Control-Request-Method: POST"
```
You should see `Access-Control-Allow-Origin: http://localhost:5173` (or 127.0.0.1:5173) in the response.
